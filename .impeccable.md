# FocusLens Visual Constraints

- Theme: dark-first native macOS styling with emerald accents on near-black surfaces.
- Accent color: `#10B981`
- Background color: `#0A0A0C`
- Typography: San Francisco for native controls, monospaced metrics where appropriate.
- Motion: subtle status pulse only; respect reduced-motion settings.
- Accessibility baseline: keyboard reachable controls, strong contrast, and descriptive labels.

## Design Context

### Users
- Solo power users who want a private, lightweight way to understand how they actually spend their working hours.
- They use FocusLens while already deep in work, so the interface needs to explain state quickly and stay out of the way.
- The main job to be done is to quietly capture activity, confirm the system is trustworthy, and make it easy to review patterns later.

### Brand Personality
- Calm
- Trustworthy
- Quietly intelligent
- Polished

### Aesthetic Direction
- Dark-first native macOS utility with restrained emerald accents and soft graphite surfaces.
- Feels like a thoughtful instrument panel, not a dashboard toy.
- Should avoid playful consumer onboarding patterns and anything that feels loud, neon, or over-decorated.

### Design Principles
- Lead with clarity: every state should explain what is happening and what the next useful action is.
- Reduce setup anxiety: first-run screens should reassure the user about privacy, locality, and what is required.
- Reveal complexity progressively: the menu bar popover should guide setup first, then surface summary and controls.
- Keep visual weight low: use quiet hierarchy, gentle contrast, and minimal motion.
- Make trust visible: reinforce local-only processing, screenshot handling, and server readiness in the UI.
